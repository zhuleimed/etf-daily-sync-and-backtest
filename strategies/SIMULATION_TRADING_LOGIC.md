# ETF 模拟盘交易逻辑与运行全流程说明

> 版本: v2.0 | 最后更新: 2026-07-06
>
> 本文档完整记录 ETF 模拟盘从数据同步到信号产生、订单执行、持仓管理、风险控制、账户核算的全套流程。
> 所有模拟盘策略共享一套框架（`simulation/framework/`），逻辑一致。
> 本文档与 `STRATEGY_COMPARISON.md`、`OPTIMIZATION_HISTORY.md` 共同构成项目的完整文档体系。

## 一、核心原则

| # | 原则 | 说明 |
|:--|:-----|:------|
| 1 | **T+1 模型** | 所有买卖决策基于 T 日收盘价数据，在 T+1 日以开盘价执行。决策和执行隔一个交易日 |
| 2 | **单ETF全仓** | 每策略每次只持有一只 ETF，全部资金（≈98%）用于买入 |
| 3 | **取消不顺延** | 涨跌停/停牌/资金不足 → 取消信号，不留存过夜。当晚由信号函数重新计算（自然恢复）|
| 4 | **信号驱动切换** | 卖出和买入都由策略信号触发。持有 ETF 时产生切换信号→明日卖出旧+买入新 |
| 5 | **风控覆盖** | 风控模式 B/C 触发时，风控产生待执行 sell 订单，**覆盖**当天信号逻辑 |
| 6 | **当日买入不可卖** | `today_opened=True` 标记保护，当日估值时跳过卖出评分。次日重置后可正常评分 |
| 7 | **最小持仓天数** | 买入后至少持有 N 天（策略默认 5~10 天），才能产生切换信号 |

## 二、时间线模型

### 2.1 交易日流程（管线步骤）

```
          T 日                              T+1 日                        T+2 日
     ═══════════════                    ═══════════════                ═══════════════

     09:30-15:00 盘中交易              09:30-15:00 盘中交易           09:30-15:00 盘中交易
     15:00 CLOSE 收盘                  15:00 CLOSE 收盘               15:00 CLOSE 收盘

20:00 cron 触发管线                  20:00 cron 触发管线              20:00 cron 触发管线
  │                                     │                                │
  ├─ Step1: 数据同步                    ├─ Step1: 数据同步                ├─ Step1: 数据同步
  │  拉取 T 日行情入库                  │  拉取 T+1 日行情入库           │  拉取 T+2 日行情入库
  │  (ETF 1514只+指数5个)              │                                 │
  │                                     │                                │
  ├─ Step2~N: 各策略模拟盘              ├─ Step2~N: 各策略模拟盘          ├─ Step2~N: 各策略模拟盘
  │  (并行子进程，顺序执行)             │                                 │
  │  ① 重置 today_opened=0              │  ① 重置 today_opened=0          │
  │     days_since_switch+=1            │     days_since_switch+=1        │
  │  ② 执行昨日待执行订单               │  ② 执行昨日待执行订单           │
  │     (用 T 日 OPEN 价)               │     (用 T+1 日 OPEN 价)        │
  │  ③ 用 T 日 CLOSE 估值              │  ③ 用 T+1 日 CLOSE 估值        │
  │  ④ 风控检查(模式B/C)               │  ④ 风控检查                    │
  │  ⑤ 信号计算 → 新待执行订单         │  ⑤ 信号计算 → 新待执行订单     │
  │  ⑥ 写入CSV + 推送微信              │  ⑥ 写入CSV + 推送微信          │
  │                                     │                                │

### 2.2 关键时序术语

| 术语 | 含义 |
|:----|:------|
| **T 日 CLOSE** | 用于计算信号和估值（决定"买什么"和"持仓值多少"） |
| **T+1 日 OPEN** | 用于执行待执行订单（决定"以什么价格成交"） |
| **T+1 日 CLOSE** | 用于估值和新的信号计算 |
| **pending_order** | 状态文件中的待执行订单，格式见 3.1 节 |
| **状态文件** | `state_{strategy_id}.json`，存储当前持仓/资金/待执行订单 |

### 2.3 完整示例（首次运行→买入→持有）

```

  ③ 信号计算 → 动量排名 #1 科创50(588000)
  ④ 空仓 → 产生买入信号 588000（明日执行）

  ① 重置 today_opened=0，days_since_switch=1

## 三、买入逻辑

### 3.1 信号来源

ETF 模拟盘仅依赖**技术指标自动计算**，无 LLM 或外部推荐。

```
动量信号（momentum_rotation）
  └─ compute_momentum_signals()
      ├─ 计算每只 ETF 的 20日动量（close/shift(20)-1）
      └─ 按动量值降序排名

复合动量（composite_momentum）
  └─ 四因子等权打分：
      ├─ 20日动量（25%）
      ├─ RPS（25%）：过去 N 日涨幅排名百分位
      ├─ 趋势强度（25%）：ADX-like 指标
      └─ 波动率调整（25%）：动量/波动率

MACD（macd_trend_rotation）
  └─ MACD(5,35,5) 信号
      ├─ DIF > DEA → 多头
      ├─ DIF ≤ DEA → 空头
      └─ MACD ≤ 0 → 强制过滤（不在多头环境不交易）

RSI（rsi_trend_rotation）
  └─ RSI(21) 非截面评分（独立评分）
      ├─ Regime分(0-2)：RSI>60=2, 50-60=1, <50=0
      ├─ 持续分(0-2)：过去10天RSI>50占比×2
      ├─ 斜率分(0-0.5)：RSI 5日变化量/10
      └─ 总得分 ≥ 2.0 才交易

ADX（adx_trend_rotation）
  └─ ADX(14) 趋势强度过滤
      ├─ ADX ≥ 25 → 趋势存在，进入动量排名
      └─ ADX < 25 → 无趋势，不交易

自适应轮动（adaptive_rotation）
  └─ detect_regime() 市场状态检测
      ├─ HS300 > MA60 × 1.03 → 牛市 → 动量模式
      ├─ HS300 < MA60 × 0.97 → 熊市 → 空仓
      └─ 中间 → 震荡 → 均值回归模式

波动率过滤（momentum_vol_filter）
  └─ 年化波动率 < 30% → 正常动量轮动
      └─ 年化波动率 ≥ 30% → 空仓

配对交易（pair_trading）
  └─ z-score 计算（3对ETF）
      ├─ 上证50↔创业板, 上证50↔科创50, 沪深300↔创业板
      └─ |z| > 2.0 → 切换到"便宜方"
```

### 3.2 信号生命周期

```
T 日 20:00（信号计算）
  → 空仓 → 买入信号写入 pending_order：{action: "buy", symbol: "588000"}
  → 状态文件保存
  ↓
T+1 日 20:00（执行）
  → 读取 pending_order
  → 开盘价检查：涨停？停牌？
  → 通过 → broker.buy() 执行 ✅
  → 不通过 → 取消，今晚重新计算 ❌
```

### 3.3 空仓时：产生买入信号

```python
if not has_position:          # 空仓
    if target_etf is not None and target_momentum > 0:
        state.pending_order = {
            "action": "buy",
            "symbol": target_etf,
            "reason": "动量信号开仓",
            "created": today_str,
        }
```

条件：
1. 当前空仓（`shares = 0`）
2. 排名第一的 ETF 有正动量（`momentum > 0`，过滤下跌趋势）

### 3.4 买入检查清单

| # | 检查项 | 条件 | 失败处理 |
|:--|:-------|:-----|:---------|
| 1 | 行情数据 | 今日 `today_data` 中有该 ETF 的行情 | `cancelled` |
| 2 | 涨停 | `open_price < prev_close × (1 + limit%)` | `cancelled` |
| 3 | 停牌 | `volume > 0`（今日有成交量） | `cancelled` |
| 4 | 涨跌停（卖出） | 卖出时检查跌停 | `cancelled` |
| 5 | 资金足够 1 手 | `cash ≥ buy_price × 100 × (1+slippage+commission)` | `cancelled` |

### 3.5 买入价格与成本计算

```
买入价 = OPEN价 × (1 + SLIPPAGE)
SLIPPAGE = 万1 = 0.0001

可用资金 = cash × ~98%（留 2% 防佣金不足）
股数 = floor(可用资金 / 买入价 / 100) × 100    ← 向下取整到整手(100股)

成本 = 股数 × 买入价
佣金 = max(成本 × 万2, 0)
总成本 = 成本 + 佣金                            ← 从现金中扣除的总额
即日均价 = 总成本 / 股数
```

示例（7/6 动量轮动买入 588000）：
```
开盘价 = 2.133
买入价 = 2.133 × 1.0001 = 2.133413
可用资金 = 10,000（全部）
股数 = int(10000 / 2.133413 / 100) × 100 = 4600
成本 = 4600 × 2.133413 = 9813.70
佣金 = 9813.70 × 0.0002 = 1.96
总成本 = 9813.70 + 1.96 = 9814.74
现金剩余 = 10000 - 9814.74 = 185.26  ✅
```

## 四、卖出逻辑

### 4.1 卖出场景

ETF 模拟盘中卖出由两种机制触发：

#### 场景 A：策略信号切换（最常见）

```
持有 588000，新信号排名第一变为 510500
  → 比较 588000 动量 vs 510500 动量
  → 差距 > MIN_SWITCH_CONVICTION(3%)？
  → 是 → 产生 switch 订单（明日卖出588000 + 买入510500）
```

执行顺序（T+1 日）：
```
① 以开盘价卖出 588000（全仓）
② 全部所得 + 剩余现金 买入 510500（全仓）
③ 切换是"原子操作"：卖出到账资金立即用于买入
```

#### 场景 B：风控触发（RISK_MODE=B 时）

```
复合动量/MACD 等策略使用风控模式 B
  ├─ 浮动止盈：从持仓最高价回落 ≥ 5% → 卖出
  ├─ ATR止损：当前价 < 成本 × (1 - 5%) → 卖出
  └─ 极端回撤：总资产从峰值回撤 ≥ 15% → 卖出

触发时 → 产生 sell 订单（覆盖当天信号逻辑）
  → 明日以开盘价执行卖出
```

#### 场景 C：平仓/止损（配对交易专用）

```
配对交易的 z-score 回归均值（|z| < close_threshold）
  → 产生 sell 订单（价差回归平仓）
或 z-score 超过 stop_threshold
  → 产生 sell 订单（止损）
```

### 4.2 卖出检查清单

| # | 检查项 | 条件 | 失败处理 |
|:--|:-------|:-----|:---------|
| 1 | 有持仓 | `shares > 0` | 跳过 |
| 2 | 跌停 | `open_price > prev_close × (1 - limit%)` | 取消标记 |
| 3 | 停牌 | `volume > 0` | 取消标记 |
| 4 | T+1保护 | `today_opened = False`（今日买入不可卖） | 跳过评分 |

### 4.3 卖出价格与成本计算

```
卖出价 = OPEN价 × (1 - SLIPPAGE)
SLIPPAGE = 万1 = 0.0001

收入 = 股数 × 卖出价
佣金 = max(收入 × 万2, 0)
净收入 = 收入 - 佣金
盈亏(PnL) = 净收入 - 持仓总成本
收益率 = 盈亏 / 持仓总成本
```

示例：
```
开盘价 = 2.116
卖出价 = 2.116 × (1 - 0.0001) = 2.115788
收入 = 4600 × 2.1158 = 9732.62
佣金 = 9732.62 × 0.0002 = 1.95
净收入 = 9732.62 - 1.95 = 9730.67
盈亏 = 9730.67 - 9814.74 = -84.07
收益率 = -84.07 / 9814.74 = -0.86%
```

### 4.4 卖出失败处理（取消不顺延）

```
跌停：开盘价 ≤ 前收盘 × (1 - limit%)
  → 取消待执行订单（不留存）
  → 今晚用当日 CLOSE 重新计算信号
  → 若趋势仍在 → 今晚重新发信号 → 明天再试
  → 若趋势已变 → 自然选其他 ETF

停牌：volume = 0
  → 同上
```

### 4.5 切换（switch）的特殊处理

切换是"卖出旧 + 买入新"的原子操作。**任一方向被封锁，整个切换取消**：

```
T 日信号：卖出588000 + 买入510500
T+1 日执行：
  ① 检查 588000 开盘 → 跌停？停牌？
     ← 是 → 整个切换取消 ❌
  ② 检查 510500 开盘 → 涨停？停牌？
     ← 是 → 整个切换取消 ❌
  ③ 全部通过 → 卖588000 + 买510500 ✅
```

## 五、T+1 保护机制（today_opened）

| 场景 | today_opened | 行为 |
|:-----|:------------:|:-----|
| T 日刚买入，T 日评估 | **True** | 跳过评分（今日不可卖）✅ |
| T+1 日评估 | **False** | 正常评分，可触发卖出/切换 |
| T+N 日评估 | **False** | 正常评分 |

**重置机制：** 每日 `run_daily()` 第 1 步：
```python
state.position.today_opened = False  # 所有持仓今日可卖
```
新买入的持仓由 `broker.buy()` 设 `today_opened = True`。

## 七、风控模式

| 模式 | 含义 | 触发效果 | 适用策略 |
|:----|:-----|:--------|:--------|
| **A** | 纯信号 | 无风控干预，完全由策略信号决定买卖 | 趋势策略（动量、ADX、RSI、自适应轮动、波动率过滤） |
| **B** | 风控全开 | 浮动止盈+ATR止损+极端回撤，产生待执行 sell 订单覆盖信号逻辑 | 信号有噪音的策略（复合动量、MACD） |

B 模式触发的风控检查：
1. **浮动止盈**：`current_price < highest_price × (1 - profit_threshold)` → 回落止盈
2. **ATR止损**：`current_price < avg_cost × (1 - stop_loss_pct)` → 硬止损
3. **极端回撤**：`total_value < peak_value × (1 - drawdown_threshold)` → 极端回撤保护

## 八、账户核算

### 8.1 现金公式（源头推导）

```python
cash = initial_capital - position.total_cost + cumulative_pnl
```

| 场景 | initial_capital | position.total_cost | cumulative_pnl | cash |
|:-----|:---------------:|:-------------------:|:--------------:|:----:|
| 初始空仓 | 10,000 | 0 | 0 | 10,000 |
| 买入 588000 | 10,000 | 9,814.74 | 0 | 185.26 |
| 卖出后空仓 | 10,000 | 0 | -84.45 | 9,915.55 |
| 再买入新ETF | 10,000 | 9,900.00 | -84.45 | 15.55 |

> **为什么用公式而非顺序减法：** 顺序减法 `cash -= cost` 会累积浮点误差，状态文件损坏后无法恢复。源头推导公式从三个源数据（初始资金、持仓成本、已平仓盈亏）推导现金，始终正确，损坏后可用公式修复。

### 8.2 总资产公式

```
stock_value = shares × current_price（今日 CLOSE）
total_value = cash + stock_value
cumulative_return = total_value / initial_capital - 1
```

### 8.3 持有天数（days_since_switch）

```
买入/切换时：days_since_switch = 0
每日递增：run_daily() 开头执行 days_since_switch += 1
估值时：不改变 days_since_switch

示例：
  7/3（周五）买入 → days_since_switch = 0
  7/6（周一）运行 → days_since_switch = 1（跳过周末）
  7/7（周二）运行 → days_since_switch = 2
  ...
  days_since_switch = 10 → 可检查切换信号
```

### 8.4 完整核算示例

```
初始资金 = 10,000

7/6 买入 588000，总成本 9,814.74
  cash = 10,000 - 9,814.74 + 0 = 185.26
  stock_value = 4,600 × 2.116 = 9,733.60
  total_value = 185.26 + 9,733.60 = 9,918.86
  cumulative_return = 9,918.86/10,000-1 = -0.81%
```

每日追加，记录完整的操作描述和持仓变化：

## 十二、组合策略（combined）

组合策略是**元策略**，不实际交易，而是读取动量轮动（80%）和配对交易（20%）的状态文件，加权计算汇总净值。

```
读取 momentum_rotation state
读取 pair_trading state

mom_return = mom_total_value / 10000 - 1
pair_return = pair_total_value / 10000 - 1

combined_total = 8000 × (1 + mom_return) + 2000 × (1 + pair_return)
combined_return = combined_total / 10000 - 1
```

## 十三、管线执行流程

### 13.1 cron 调度

```
0 20 * * 1-5                     ← 交易日 20:00（周一至周五）
PYTHONUNBUFFERED=1                ← 无缓冲输出，日志按时间顺序
python pipeline.py                ← 管线主入口
>> logs/pipeline_YYYYMMDD.log     ← 日志文件
```

### 13.2 pipeline 步骤

```
pipeline.py
  │
  ├─ Step 1: 数据同步（main.py --sync-only）         [必需]
  │   拉取 ETF 日线 + 指数日线 → 写入 etf_daily.db
  │
  ├─ Step 2: 动量轮动模拟盘（-m momentum_rotation.daily）  [必需]
  │
  ├─ Step 3~9: 其余策略模拟盘（可选，一个接一个）
  │   复合动量 → MACD趋势 → RSI趋势 → 自适应轮动
  │   → ADX趋势 → 波动率过滤 → 配对交易
  │
  ├─ Step 10: 组合策略模拟盘（可选，依赖前两者）
  │
  └─ 推送：管线汇总 + 策略汇总日报
```

### 13.3 代码调用链

```
pipeline.py
  └─ subprocess.run([python, "-m", "simulation.strategies.xxx.daily"])
        └─ main()
             ├─ is_trading_day()              → 交易日判断
             ├─ get_latest_trading_day()      → 数据就绪检查
             ├─ load_latest_data()            → 从 etf_daily.db 加载数据
             ├─ StateManager.load()           → 加载/初始化 JSON 状态
             ├─ SimBroker()                   → 交易模拟器（滑点+佣金）
             ├─ DailySimEngine.run_daily()    → 每日流程引擎
             │    ├─ reset today_opened       → 重置 T+1 保护
             │    ├─ _execute_pending_order() → 执行昨日订单
             │    ├─ 风控检查(模式B)          → 止损/止盈
             │    ├─ 信号计算                 → 产生新待执行订单
             │    └─ sim_db 快照              → 写入 SQLite 日结
             ├─ append_simulation_log()       → 写入 CSV 日志
             └─ push_daily_report()           → 推送微信日报
```

## 十四、异常场景汇总

| 场景 | 处理方式 | 订单状态 | 后续 |
|:-----|:---------|:---------|:------|
| 买入时涨停 | 取消信号 | `pending_order` 清空 | 今晚重新计算信号 |
| 买入时停牌 | 取消信号 | `pending_order` 清空 | 同上 |
| 卖出时跌停 | 取消信号 | `pending_order` 清空 | 今晚重新计算信号 |
| 卖出时停牌 | 取消信号 | `pending_order` 清空 | 同上 |
| 切换时任一方向封锁 | 整个切换取消 | `pending_order` 清空 | 同上 |
| 资金不够 1 手 | 取消信号 | `pending_order` 清空 | 等高价位 ETF 回落或换标的 |
| 风控 B 触发 | 产生 sell 订单覆盖信号 | `pending_order={sell}` | 次日开盘卖出 |
| 非交易日（周末/节假日） | 整条管线跳过 | 不变 | 下一交易日恢复 |
| 最新数据非当日 | 跳过该策略，发出警告 | 不变 | 等待数据同步完成 |
