# 模拟盘运行买卖逻辑

> 本文档描述模拟盘（paper trading）从数据接收、信号产生、订单执行到持仓管理的完整流程。
> 所有模拟盘策略共享一套框架（`simulation/framework/`），逻辑一致。

---

## 1. 时间线（T+1 模型）

```
T日 20:00（数据同步 → 管线启动）
  │  ┌─ 非交易日 → 跳过，不执行任何操作
  │  └─ 交易日 →
  │      1. 用 T日收盘价（close）计算信号
  │      2. 产生"待执行订单"（买入/卖出/切换）
  │      3. 保存到 state_{id}.json
  │      ── 今日不交易，订单明日执行 ──
  │
T+1日 20:00
  │  ┌─ 非交易日 → 跳过
  │  └─ 交易日 →
  │      1. 用 T+1日开盘价（open）执行昨日待执行订单
  │         → 检查涨跌停/停牌
  │      2. 用 T+1日收盘价（close）算新信号
  │      3. 产生新的待执行订单...
```

**定时机制：** cron 在交易日 20:00 触发 `pipeline.py`，非交易日由 `is_trading_day()` 判断跳过。

## 2. 首次运行

首次运行时 `state_{id}.json` 不存在：

```
state_mgr.load() → 返回 None
state_mgr.init_new(initial_capital) → 新建状态，资金 = 初始资金，空仓
```

**空仓状态下，信号函数只会产生"买入信号"，不可能有卖出信号。**

买入信号示例：
```json
{"action": "buy", "symbol": "159915", "reason": "动量信号开仓", "created": "2026-07-01"}
```

## 3. 订单执行

### 3.1 执行顺序

每天第一步执行昨日的待执行订单。三种订单类型：

| 订单类型 | 说明 | 执行方式 |
|:--------|:-----|:--------|
| `buy` | 买入目标ETF | 用今日开盘价买入，全部资金（扣除佣金） |
| `sell` | 卖出持仓ETF | 用今日开盘价卖出持仓的全部股数 |
| `switch` | 卖出旧+买入新 | **先**用今日开盘价卖出旧ETF → 全部所得**再**买新ETF |

**切换是原子操作**：卖出和买入在同一交易日完成，卖出到账资金立即用于买入。

### 3.2 资金分配

**单ETF全仓模式：** 全部资金（约98%，留2%防佣金不足）用于买入唯一的持仓ETF。

```
初始资金 10,000 元
买入信号 → 买入 floor(10,000 × 0.98 / 开盘价 / 100) × 100 股
切换信号 → 全部卖出 → 全部所得买入新ETF
```

不涉及多ETF资金平分。

## 4. 封锁处理（涨跌停/停牌）

### 4.1 检查规则

| 条件 | 判定 | 限制 |
|:----|:-----|:-----|
| 涨停 | `open >= prev_close × (1 + limit%)` | 不能买入 |
| 跌停 | `open <= prev_close × (1 - limit%)` | 不能卖出 |
| 停牌 | `volume == 0` | 买卖都不能执行 |

其中 `limit%`：
- 上证50/沪深300/中证500/1000/2000 → 10%
- 创业板(159915)/科创板(588000) → 20%

### 4.2 被封锁后：取消，不顺延

```
T日: 科创50 → 买入信号（明日执行）
T+1日: 科创50开盘涨停 → 买入被封锁 → 订单丢弃

       ── 当晚重新用 T+1日数据算信号 ──
       │
       ├─ 如果科创50趋势仍在 → 今晚继续发买入信号
       │  → 明天再试一次（效果=可接受的重试）
       │
       └─ 如果趋势已变（涨停打开/回落）→ 今晚自然选其他ETF
          → 比硬性顺延旧订单更合理
```

**为什么不顺延：** ETF很少连续涨停，且旧信号的买入理由（动量/趋势/均值回归）在几天后可能已失效。放弃后当晚重新算信号，让策略自适应最新市场状态。

## 5. 持仓锁定

### 5.1 不重复买入

```
if not has_position:          # 只有空仓才产生买入信号
    产生买入信号(...)
```

已有持仓时，新产生的信号只能是 `hold`（持有）或 `switch`（切换），不会是 `buy`。

### 5.2 当日买入不可卖（T+1制度）

```python
state.position.today_opened = True  # 标记：今天刚买入
# 即使晚上信号说"卖出"，也被此标记阻止
# 第二天重置：state.position.today_opened = False
```

### 5.3 最小持仓天数

```python
if days_since_switch < min_hold_days:
    # 不检查切换，继续持有
```

各策略默认值：5~10天。买入后至少持有这段时间才能切换。

## 6. 风控模式

| 模式 | 含义 | 触发效果 | 适用策略 |
|:----|:-----|:--------|:--------|
| **A** | 纯信号 | 无风控干预 | 趋势策略（动量、ADX、RSI、自适应轮动、波动率过滤） |
| **B** | 风控全开 | 浮动止盈+ATR止损+极端回撤 | 信号有噪音的策略（复合动量、MACD） |

B模式触发时，风控产生待执行 `sell` 订单，覆盖当天信号逻辑。

## 7. 状态文件（state_{id}.json）

每个策略一个状态文件，位于 `simulation/output/state_{strategy_id}.json`：

```json
{
  "version": 4,
  "last_update": "2026-07-01",
  "cash": 1200.00,
  "initial_capital": 10000.00,
  "position": {
    "symbol": "159915",
    "shares": 1500,
    "avg_cost": 2.3580,
    "highest_price": 2.4500
  },
  "days_since_switch": 45,
  "peak_value": 22555.00,
  "pending_order": null,
  "strategy_name": "momentum_rotation",
  "total_value": 22555.00
}
```

**保护措施：**
- `.gitignore` 已排除 `simulation/output/`，git不会误删
- `state.py` 使用原子写入（`tempfile` + `os.replace`），断电不损坏
- CSV日志（`sim_log_{id}.csv`）有完整历史，可作为恢复依据

## 8. 交易成本

| 项目 | 费率 | 说明 |
|:----|:----|:-----|
| 佣金 | 万2（0.02%） | 买入卖出各收一次 |
| 滑点 | 万1（0.01%） | 买：`open×(1+0.01%)`，卖：`open×(1-0.01%)` |
| 印花税 | 0 | ETF免印花税 |

**一次切换（买入+卖出）的总成本 ≈ 0.06%**。

## 9. 汇总日报

管线运行结束后，`summary.py` 自动读取所有 `state_{id}.json` 和 `sim_log_{id}.csv`：
- 计算各策略累计/年化收益、夏普、最大回撤、日胜率
- 一条微信推送全部策略横向对比
- 新策略自动发现，无需配置

## 附录：代码调用链

```
pipeline.py
  └─ subprocess.run([python, "-m", "simulation.strategies.xxx.daily"])
        └─ main()
             ├─ is_trading_day()             → 交易日判断
             ├─ get_latest_trading_day()     → 数据就绪检查
             ├─ load_latest_data()           → 从 etf_daily.db 加载数据
             ├─ StateManager()               → 加载/初始化状态
             ├─ SimBroker()                  → 交易模拟器
             ├─ DailySimEngine()             → 每日流程引擎
             │    ├─ _execute_pending_order()  → 执行昨日订单
             │    ├─ 风控检查                  → 止损/止盈
             │    └─ 信号计算                  → 产生新订单
             ├─ append_simulation_log()      → 写入CSV日志
             └─ push_daily_report()          → 推送微信日报
```
