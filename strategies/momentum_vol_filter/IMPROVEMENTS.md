# 策略改进建议

基于回测分析发现的逻辑缺陷，提出以下改进方案。按实施难度和预期收益排序。

---

## ✅ 已实施：最小切换置信度 (MIN_SWITCH_CONVICTION)

```python
# config.py
MIN_SWITCH_CONVICTION = 0.02   # 动量差距≥2%才切换
```

**效果**: 切换次数减少 58%，全周期收益持平。普涨市中追涨杀跌减轻。

---

## 🔧 建议改进（按优先级排列）

### 1. TOP-N 持仓（推荐程度：⭐⭐⭐⭐⭐）

**问题**: 当前只持有一只 ETF，排第 1 和第 5 的差异可能很小。一旦切换错就完全踏空。

**改进**: 持有动量排名前 N 只 ETF（如 N=2），等权或按动量加权分配资金。

**优点**:
- 减少单只 ETF 的择时风险
- 自然降低切换频率（前 2 名比第 1 名更稳定）
- 2025 年普涨市中不会完全踏空

**实现思路**:
```python
# 每期持有 top-2，每只 50%
targets = ranking.head(2)  # 排名前2
for sym in targets:
    target_weight = 0.5
    # 调整持仓至目标权重
```

**预估效果**: 2025 年普涨市从 -10% 提升至 +5%~+10%

---

### 2. 动态动量窗口（推荐程度：⭐⭐⭐⭐）

**问题**: 固定 15 日窗口在所有市场环境中一样——震荡市中太慢、趋势市中太快。

**改进**: 根据市场波动率（VIX/ATR）或趋势强度动态调整动量窗口。趋势强时用长窗口（稳），趋势弱时用短窗口（敏）。

**实现思路**:
```python
# 当各ETF动量值方差大（分化显著）→ 短窗口（快速捕捉）
# 当各ETF动量值方差小（普涨/普跌）→ 长窗口（减少噪音）
momentum_spread = momentum.std()  # 各ETF动量值的标准差
if momentum_spread > 0.03:
    window = 10   # 分化市：快速反应
else:
    window = 20   # 同步市：稳住别动
```

**预估效果**: 在分化和普涨市之间自适应，平滑年度收益波动

---

### 3. 动量质量打分（推荐程度：⭐⭐⭐）

**问题**: 当前只看 15 日总收益率，未考虑动量质量和动能衰减。

**改进**: 综合多个维度的评分：
- 短期动量（5 日）
- 中期动量（15 日）
- 动量加速度（动量的一阶差分）
- 价格相对于移动平均线的位置

**实现思路**:
```python
score = (mom_5d * 0.3 + mom_15d * 0.5 + mom_accel * 0.2)
# mom_accel = mom_5d - mom_15d  # 动量加速度（正在加速的优先）
```

---

### 4. 市场状态感知开仓（推荐程度：⭐⭐⭐）

**问题**: 当前的空仓条件只有"最强动量 ≤ 0"——太简单。在熊市中虽能避险，但在牛市中跑输。

**改进**: 增加市场状态判断来决定仓位水平：
- **趋势市**: 满仓持有动量第 1 名
- **震荡市**: 半仓或持有动量前 2 名
- **下跌市**: 空仓（维持当前逻辑）

可通过沪深300的 20 日波动率或 ADX 指标判断市场状态。

---

### 5. 持仓冷却期（推荐程度：⭐⭐）

**问题**: 刚买入一只 ETF，如果第二天动量排名变了就又卖出。频繁微调产生大量噪音交易。

**改进**: 每次切换后至少持有 N 个交易日才能再次切换。

**实现思路**:
```python
if self.days_since_last_switch < MIN_HOLD_DAYS:
    return  # 不切换，继续观察
```

---

### 6. 趋势强度过滤（推荐程度：⭐⭐）

**问题**: 动量轮动在大盘方向性明确时表现最好——无论涨跌，只要方向一致。但在横盘震荡中切换频率最高、表现最差。

**改进**: 引入趋势强度指标（如 ADX），只有当趋势强度高于阈值时才执行轮动。横盘市中减少干预。

---

## 改进优先级建议

```
第一优先   TOP-N 持仓       → 解决普涨踏空，效果最直接
第二优先   动态动量窗口      → 自适应不同市场
第三优先   动量质量打分      → 提升选标的质量
第四优先   市场状态感知      → 宏观仓位管理
第五优先   持仓冷却期        → 减少噪音交易
```

## 建议的实施路线

1. 先做 **TOP-N 持仓** + **最小置信阈值**（已完成），这两个改动最小、效果最确定
2. 回测验证后加入**动态动量窗口**，需要充分测试不同参数组合
3. 最后考虑**动量质量打分**和**市场状态感知**，这两个需要更多工程投入
