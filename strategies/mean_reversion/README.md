## 策略概述

**核心思想**：A 股宽基指数之间存在明显的动量轮动效应——资金在不同市值风格（大盘/中盘/小盘/微盘）之间切换，强势风格在一段时间内会持续跑赢弱势风格。本策略通过每日计算 5 只宽基 ETF 的 N 日动量（N 日收益率），买入动量最强的标的，在最强势的风格上获取超额收益。

**策略标的池**（5 只宽基 ETF，覆盖 A 股主要市值风格）：

| 代码 | 名称 | 跟踪指数 | 风格 |
|:----:|------|---------|:----:|
| 510050 | 上证50ETF（华夏） | 上证50 | 超大盘 |
| 510300 | 沪深300ETF（华泰柏瑞） | 沪深300 | 大盘 |
| 510500 | 中证500ETF（南方） | 中证500 | 中盘 |
| 512100 | 中证1000ETF（南方） | 中证1000 | 小盘 |
| 563000 | 中证2000ETF（华夏） | 中证2000 | 微盘 |

**决策流程**（每个交易日）：
```
行情数据 → 计算5只ETF的20日动量 → 动量降序排序 →
检验摩擦成本 → 如需切换：启动5日渐进调仓 → 记录当日账户状态
```

## 模块详解

### config.py — 策略配置

所有策略参数集中管理，**修改参数后直接生效，无需改动代码逻辑**。

```python
# ETF 标的池
ETF_POOL = {"510050": "上证50ETF（华夏）", ...}   # 5 只宽基 ETF
ETF_SYMBOLS = list(ETF_POOL.keys())                # 代码列表

# 回测参数
INITIAL_CAPITAL = 10000       # 初始资金（元）
START_DATE = "2024-01-01"     # 回测开始日期
END_DATE = ""                 # 空字符串 = 到最新数据

# 动量信号参数
MOMENTUM_WINDOW = 20          # N日动量窗口（默认20个交易日）

# 交易费用（ETF特有：无印花税）
COMMISSION_RATE = 0.0002      # 佣金万分之二（0.02%）
SLIPPAGE = 0.0001             # 滑点万分之一（0.01%）
TAX_RATE = 0.0                # 印花税 0%（ETF 免收）

# 冲击成本（按流动性动态计算）
IMPACT_COST_COEF = 0.1        # 冲击系数

# 渐进调仓参数
ADJUSTMENT_DAYS = 5           # 调仓周期（分5个交易日完成）

# 风控参数
STOP_PROFIT_THRESHOLD = 0.10  # 浮动止盈激活阈值
STOP_PROFIT_DRAWBACK = 0.05   # 浮动止盈回撤阈值
ATR_MULTIPLIER = 4.0          # ATR 止损倍数
MAX_DRAWDOWN = 0.15           # 极端回撤阈值

# 风控模式
RISK_MODE = "A"               # A=纯信号, B=全开, C=仅极端回撤
```

### data.py — 数据加载

**职责**：从 SQLite 数据库加载 5 只 ETF 的日线数据，做日期对齐，预计算辅助列。

**数据流**：
```
SQLite (etf_daily.db)
        │
        ▼
_load_single_etf()     ← 读取单只 ETF（OHLCV + 日期范围过滤）
        │
        ▼
  计算辅助列：
  ├── pct_chg           ← 日收益率 close.pct_change()
  ├── cumulative_returns ← 累计收益 (1+pct_chg).cumprod()
  ├── amount            ← 成交额 close × volume
  ├── amount_ma20       ← 20日均成交额（冲击成本用）
  ├── TR / ATR          ← True Range / Average True Range
  └── momentum          ← N日动量 close / close.shift(N) - 1
        │
        ▼
load_all_etf_data()
  ├── 数据前载（扩展起始日 = start_date - 动量窗口×3天，确保动量计算有足够历史数据）
  ├── 日期对齐（取所有 ETF 共同交易日）
  └── 裁剪回 start_date
```

**关键细节**：
- `momentum` 列使用 `shift(20)` 向后看 20 天，**无前视偏差**
- ATR 使用 `rolling(20).mean().bfill()` 计算，无未来数据
- 为了确保回测第一天就能计算动量，数据前载了动量窗口×3 个自然日

### signal.py — 动量信号

**核心逻辑**：
1. 每日读取所有 ETF 预计算的 `momentum` 值（当前收盘价 ÷ 20 天前收盘价 - 1）
2. 按动量值降序排列，排名第 1 的即为"最强标的"
3. 将排名结果传递给引擎做交易决策

**三个关键阈值条件**（在 engine 中实现，signal 只负责计算和排序）：
- **无持仓** → 仅当最强标的动量 > 0 时开仓
- **有持仓且最强标的不同** → 仅在预期超额收益 > 摩擦成本时切换
- **有持仓且最强标的就是当前持仓** → 继续持有

### cost.py — 摩擦成本

**三种成本构成**：

| 成本类型 | 计算公式 | 典型值 |
|---------|---------|:------:|
| 佣金 | `交易金额 × COMMISSION_RATE` | 0.02% |
| 滑点 | `交易金额 × SLIPPAGE` | 0.01% |
| 冲击成本 | `交易金额 × (调仓金额/日均成交额 × IMPACT_COST_COEF)` | 动态计算 |

**调仓决策校验**：
```
if 目标动量 - 当前动量 > 双边摩擦成本比率:
    执行切换
else:
    保持当前持仓
```

冲击成本与 ETF 流动性成反比——成交额越大的 ETF 冲击成本越低，这自动抑制了在流动性差的品种上的频繁交易。

### risk.py — 风控模块

三种风控机制，执行优先级：**极端回撤 > ATR 止损 > 浮动止盈**。

#### 1. 浮动止盈（Trailing Stop-Profit）
- **激活条件**：持仓收益（按收盘价计算）≥ `STOP_PROFIT_THRESHOLD`（10%）
- **触发条件**：激活后，日内最低价从最高点回撤 ≥ `STOP_PROFIT_DRAWBACK`（5%）时全部卖出
- 使用 `high` 更新峰值（追踪日内最高点），用 `low` 做回撤检测（更贴合实盘）

#### 2. ATR 动态止损
- **止损线** = 持仓成本价 - `ATR_MULTIPLIER` × 当日 ATR
- **触发条件**：日内最低价跌破止损线
- ATR 值随市场波动率自动调整——波动大时止损线更宽，波动小时更紧

#### 3. 极端回撤（账户级兜底）
- **触发条件**：总资产从最高点回撤 ≥ `MAX_DRAWDOWN`（15%）
- 触发后将 `peak_total_value` 重置为当前现金值，防止死亡螺旋

**风控检测使用 `low` 价格而非 `close` 价格**：
- 实盘中止盈/止损单是**价格触碰即执行**，不等到收盘
- 使用 `low` 更贴近实盘，但也导致风控触发更频繁

### engine.py — 回测引擎

**逐日事件驱动主循环**（每个交易日执行一次）：

```
for idx in range(n):   ← n = 总交易日数
    │
    ├─ Step 1: 获取当日行情 (today_data)
    │
    ├─ Step 2: 风控检查 (非A模式)
    │   ├── 更新峰值（high价）
    │   ├── 更新账户最高市值
    │   ├── 极端回撤检查 → 触发则清仓并记录
    │   ├── ATR 止损检查 → 触发则卖出
    │   └── 浮动止盈检查 → 触发则卖出
    │
    ├─ Step 3: 执行渐进调仓（如有进行中的调仓）
    │
    ├─ Step 4: 计算动量信号 & 排名
    │
    ├─ Step 5: 交易决策
    │   ├── 无持仓 → 动量 > 0 则开仓
    │   └── 有持仓 → 摩擦成本校验 → 启动调仓或维持
    │
    └─ Step 6: 记录当日账户状态
```

**渐进调仓**：
- 默认分 5 天完成，每天卖出旧标的 1/5、买入新标的
- 减少对市场的冲击，也避免单日大量交易带来的滑点损失

**FIFO 成本核算**：
- 每笔买入记录为 BuyLot（日期、份额、价格、成本）
- 卖出时按先进先出（FIFO）匹配买入批次，计算持仓天数和盈亏
- 这确保了多批次买入的平均成本计算准确

| 类别 | 指标 | 计算方式 |

| 收益 | 年化收益率 | (1+total)^(252/n) - 1 |
| 收益 | 沪深300超额 | 策略收益 - 沪深300收益 |
| 收益 | 等权超额 | 策略收益 - 5只等权组合收益 |
| 风险 | 最大回撤 | 净值从峰值到谷底的最大跌幅 |
| 风险 | 回撤持续天数 | 从首次负回撤到恢复前高的天数 |
| 风险 | 年化波动率 | 日收益率标准差 × √252 |

## 风控模式说明

| 模式 | 名称 | 止损 | 止盈 | 极端回撤 | 适用场景 |
|:----:|:----:|:----:|:----:|:--------:|---------|
| **A** | 纯信号 | ✗ | ✗ | ✗ | 追求最高收益，接受较大回撤 |
| **B** | 全开 | ✓ | ✓ | ✓ | 保守风格，参数由 config 控制 |
| **C** | 兜底 | ✗ | ✗ | ✓ | 仅防极端风险 |

**设置方式**：
```bash
# 方法1：修改 config.py
RISK_MODE = "B"

# 方法2：命令行覆盖
python -m strategies.momentum_rotation.run --risk-mode C
```

### 为什么纯信号（A）通常比带风控（B）收益更高？

详细分析见本文件末尾的[回测结果参考](#回测结果参考)部分。核心原因：

1. **动量轮动策略本质是趋势跟踪**——收益来源于完整持有上涨趋势
2. **止损的逆趋势操作打断了上涨趋势**——止损后常错过后续反弹
3. **止盈在高点附近卖出**——切断了最大盈利的持仓
4. **风控的累积损耗**——频繁微小亏损的复合效应远超其保护价值

## 策略逻辑详解

### 动量信号

```python
# 第 t 日的 N 日动量
momentum(t) = close(t) / close(t - N) - 1
```

- N = 20（默认），即过去 20 个交易日的累计收益率
- 每日重新计算，仅使用当日及之前的数据，**无前视偏差**
- 动量 > 0 代表过去 N 日上涨，< 0 代表过去 N 日下跌

### 开仓条件

只有在**最强标的动量 > 0** 时才开仓。这意味着：
- 全市场下跌时，策略保持空仓（现金）
- 市场首次出现正动量时才入场
- 这天然规避了下跌趋势，但也可能错过反弹初段

### 调仓决策

```
if 目标ETF动量 - 当前ETF动量 > 双边摩擦成本:
    启动渐进调仓（分 N 天切换到目标ETF）
else:
    继续持有当前ETF
```

摩擦成本校验防止因为微小的动量差异而频繁交易。

### 渐进调仓

- 默认分 5 个交易日完成切换
- 每日卖出旧标的 1/N，买入新标的
- 减少市场冲击，平滑切换成本

### 交易执行

- 买入价 = 当日收盘价 × (1 + 滑点)
- 卖出价 = 当日收盘价 × (1 - 滑点)
- 买入份额取整为 100 的倍数（A 股市场最小交易单位）
- FIFO 成本核算：多批次买入时，卖出按先进先出匹配成本

数据通过 `etf_sync/` 顶层模块从腾讯财经 API（`web.ifzq.gtimg.cn`）获取，新浪 API 作为备选。

### 数据范围

- **ETF 日线**：2023-03-02 至最新（约 800 行/只）
- **指数日线**：2024-01-02 至最新

## 已知问题与注意事项

### 1. 信号与执行同日（轻微前瞻偏差）

策略使用**当日收盘价同时计算动量信号和执行交易**。在实盘中，信号计算和执行之间有一段时间差。这属于行业常见做法，对 20 日动量策略影响极小（<0.5%）。

如需消除，可将 `data.py` 中的动量计算改为：
```python
# 更保守：用昨日收盘价计算动量
df["momentum"] = df["close"].shift(1) / df["close"].shift(MOMENTUM_WINDOW + 1) - 1
```

### 2. 风控使用 low 价导致更频繁触发

风控使用日内 `low` 价而非 `close` 价触发，这在实盘中更真实（止损单是价格触碰即执行），但在回测中会让风控看起来比同期 close 版本更差。这是**正确性提升**而非 bug。

### 3. 基准指数数据起始较晚（2024-01）

沪深300 指数数据从 2024-01-02 开始，若回测起始日期早于此，基准收益率为空。

### 4. 无数据时的行为

- 若某 ETF 数据不足动量窗口长度，该 ETF 被排除
- 若所有 ETF 均被排除，抛出 `ValueError`
- 共同交易日为空时回测无法运行

### 5. 虚拟卖出

回测结束时若有未平仓持仓，以最后一日收盘价虚拟卖出计算盈亏，不计佣金和税费。

### 6. 死亡螺旋（已修复）

早期版本中，极端回撤触发后 `peak_total_value` 未重置，导致后续每次开仓都立即再次触发极端回撤，形成死亡螺旋。当前版本在极端回撤触发后将峰值重置为当前现金值。

## 回测结果参考

以下为 2024-01 至 2026-06 期间回测结果（仅供参考，不代表未来表现）：

### 三种风控模式对比（全区间）

| 指标 | A 纯信号 | B 全风控(ATR=4) | C 仅极端回撤 |
|:----:|:--------:|:--------------:|:----------:|
| 累计收益率 | **67.89%** | 29.87% | ~60%+ |
| 年化收益率 | **24.54%** | 11.71% | ~22% |
| 最大回撤 | -15.05% | -20.07% | ~-16% |
| 夏普比率 | 0.996 | 0.508 | ~0.9 |
| 最终资金 | **16,789** | 12,987 | ~16,000 |

### 不同市场环境下的表现

| 市场环境 | 区间 | 沪深300 | A 纯信号 | B 全风控 |
|:-------:|:----:|:------:|:--------:|:--------:|
| 熊/震荡 | 2023/3→2024/12 | -1.30% | **+8.94%** | -22.34% |
| 牛市 | 2024/1→2026/6 | +49.41% | **+67.89%** | +29.87% |

### 核心发现

1. **A 纯信号在两种市场中均跑赢沪深300**——动量轮动策略的选股逻辑有效
2. **B 全风控在两种市场中均跑输 A**——风控打断了动量策略的盈利来源
3. 风控在熊市中放大了亏损（反复止损），在牛市中削减了收益（打断趋势）
4. 对于趋势跟踪类的动量轮动策略，传统价格止损/止盈与策略逻辑自相矛盾
5. ATR=4.0 比默认的 2.0 效果更好（更宽松的止损减少了误触发）
